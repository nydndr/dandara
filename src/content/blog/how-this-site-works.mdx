## The why

- **If I don't land any new opportunities by the end of 2026, this website has failed.** This website was built to market me.
- **If I don't learn any new skills or concepts in the making of this, this website has failed.**
- **If this website can't side by side with those of people I admire, this website has failed.**

## Futurelog

For future optimizations I would like to do on the website I tried to scope them in a way that I can bash one in a session of work.

I don't have any obligation to do this, but some days I get an itch to code and these should be fun enough to research, code, test and publish in said one day. Bigger things should be split into smaller parts to keep this concept working.

- <ChangelogBadge type="add" done={false} /> **Add social cards for blog
  posts.** When a post is shared the link creates a beautiful and dynamic image
  to accompny it. It's [those
  cards](https://x.com/jasonlbeggs/status/1982801017822466560?s=20) you spot on
  your twitter timeline.
- <ChangelogBadge type="add" done={false} /> **Add tag support for articles.**
  This is one way to allow for filtering and to offer similar reads at the end
  of the article - check an example of this at the bottom of [Brian Lovin's blog
  posts](https://brianlovin.com/writing/texting-with-notion-ai).
- <ChangelogBadge type="change" done={false} /> **Refactor Projects page to work
  with MDX files.** I experimented doing with JSON just to see the benefits of
  each approach, but I don't think it was the best. Doing it with MDX would also
  allow for having individual pages for each project down the line.
- <ChangelogBadge type="learn" done={false} /> **Tailwind v4**. It's been a long
  time since I actually sat down to understand how Tailwind is spinning and
  maybe I could be using it in smarter ways. We'll see!
- <ChangelogBadge type="learn" done={false} /> **Next 16**. Next.js has such
  great documentation that you can easily get away with assembling things
  without understanding *a lot*. I don't like that feeling though.
- <ChangelogBadge type="change" done={false} /> **Refactor css** code.
- <ChangelogBadge type="add" done={false} /> **Add cumulative rendering to
  [Now](/now) page.** Not many people do this, but I would like the Now page to
  work like a "work journal" down the line. Think something like what [Maggie
  Appleton Now page](https://maggieappleton.com/now/).
